## 一、业务背景

教育类私域销转业务：通过抖音/小红书/视频号/天猫等渠道引流到企微私域，按"期"开营（目前到第107期），推广员在社群做转化（直播转化 + 追单），销售英语课程。

**三个 SKU**：学霸陪跑营、阅读强化营、自拼训练营

**组织架构**：城市(杭州/长沙) → 分区负责人 → 组长 → 推广员(主R)

**核心漏斗**：例子数 → D1到课 → D2到课 → D3到课 → 直播转化 → 追单转化

```
│   └── meta.json           # 拉取时间、数据版本等元信息
├── fetch-data.sh       # 一键数据拉取脚本
└── docs/
    └── superpowers/
        └── specs/
            └── 2026-04-06-sales-dashboard-design.md
```

**技术选型**：
- 纯前端：HTML + CSS + Vanilla JS（无框架，单页应用）
- 图表：ECharts 5.x（CDN）
- 字体：DM Sans + Noto Sans SC（CDN）
- 本地服务：`python3 -m http.server 8891` 或 `npx serve`
- 数据拉取：Node.js 脚本调用飞书 Bitable API

**为什么不用框架**：
- 单页看板，复杂度不需要 React/Vue
- 无构建步骤，改完刷新即见
- 迭代快，加图表加页面直接写

### 4.3 视图一：概览 Dashboard

**KPI 卡片行（6个）**：
| 指标 | 说明 |
|---|---|

**图表区**：
1. **GMV 月度趋势**（折线图，全宽）— 按 SKU 分色，可切换合计/分SKU
2. **转化率趋势**（折线图）— 总转率、直转率、追单率三条线

4. **SKU GMV 占比**（堆叠柱状图）— 按月显示三个 SKU 的 GMV 构成

**智能分析区**：

## 六、数据拉取方案

### fetch-data.sh 脚本逻辑

```
1. 调用飞书 Bitable API，分页拉取5张表的全量数据
2. 清洗数据：
   - 提取 formula/lookup 字段的实际值（飞书返回 {type:2, value:[xxx]} 格式）
   - 时间戳转为可读日期
   - 空值处理
3. 按表写入 data/ 目录下的 JSON 文件
4. 写入 meta.json（拉取时间、各表记录数）
5. 输出摘要
```

### 数据更新流程
- 手动：终端运行 `./fetch-data.sh`
- 自动：可配置 cron 定时执行（如每小时一次）
- 看板打开时显示"数据更新时间"，一眼看出数据是否最新
